Class SC.ScrollerView

Extends: SC.View
Defined in: frameworks/sproutcore/frameworks/desktop/views/scroller.js
Availability: since SproutCore 1.0

Overview

Displays a horizontal or vertical scroller. You will not usually need to work with scroller views directly, but you may override this class to implement your own custom scrollers.

Because the scroller uses the dimensions of its constituent elements to calculate layout, you may need to override the default display metrics.

You can either create a subclass of ScrollerView with the new values, or provide your own in your theme:

SC.mixin(SC.ScrollerView.prototype, {
scrollbarThickness: 14,
capLength: 18,
capOverlap: 14,
buttonOverlap: 11,
buttonLength: 41
});

You can change whether scroll buttons are displayed by setting the hasButtons property.

Property Details

buttonLength [R/W]

The width or height of the up/down or left/right arrow buttons.

-   Number buttonLength

Discussion

If the scroller is not displaying arrows, this is the width or height of the end cap.

buttonOverlap [R/W]

The amount by which the thumb overlaps the arrow buttons.

-   Number buttonOverlap

Discussion

If the scroller is not displaying arrows, this is the amount by which the thumb overlaps the end cap.

capLength [R/W]

-   Number capLength

capOverlap [R/W]

The amount by which the thumb overlaps the cap.

-   Number capOverlap

hasButtons [R/W]

Whether or not the scroller should display scroll buttons

-   Boolean hasButtons

Default Value: YES

isEnabled [R/W]

YES to enable scrollbar, NO to disable it.

-   Object isEnabled

Discussion

Scrollbars will automatically disable if the maximum scroll width does not exceed their capacity.

layoutDirection [R/W]

Determine the layout direction.

-   Object layoutDirection

Discussion

Determines whether the scrollbar should appear horizontal or vertical. This must be set when the view is created. Changing this once the view has been created will have no effect.

maximum [R/W]

The maximum offset value for the scroller.

-   Number maximum

Discussion

This will be used to calculate the internal height/width of the scroller itself.

When set less than the height of the scroller, the scroller is disabled.

minimum [R/W]

The minimum offset value for the scroller.

-   Number minimum

Discussion

This will be used to calculate the internal height/width of the scroller itself.

proportion [R/W]

The portion of the track that the thumb should fill.

-   Number proportion

Discussion

Usually the proportion will be the ratio of the size of the scroll view's content view to the size of the scroll view.

Should be specified as a value between 0.0 (minimal size) and 1.0 (fills the slot). Note that if the proportion is 1.0 then the control will be disabled.

scrollbarThickness [R/W]

The width (if vertical scroller) or height (if horizontal scroller) of the scrollbar.

-   Number scrollbarThickness

shouldScrollToClick [R/W]

If YES, a click on the track will cause the scrollbar to scroll to that position.

-   Object shouldScrollToClick

Discussion

Otherwise, a click on the track will cause a page down.

In either case, alt-clicks will perform the opposite behavior.

value [R/W]

The value of the scroller.

-   Number value

Discussion

The value represents the position of the scroller's thumb.
