Namespace SC.Validatable

Defined in: frameworks/sproutcore/frameworks/foundation/mixins/validatable.js

Overview

Views that include the Validatable mixin can be used with validators to ensure their values are valid.

Property Details

errorLabel [R/W]

This property must return the human readable name you want used when describing an error condition.

+   String errorLabel

Discussion

For example, if set this property to "Your Email", then the returned error string might be something like "Your Email is not valid".

You can return a loc string here if you like. It will be localized when it is placed into the error string.

isValid [R/W]

YES if the receiver is currently valid.

+   Boolean isValid

Discussion

This property watches the value property by default. You can override this property if you want to use some other method to calculate the current valid state.

ownerForm [R/W]

The form that the view belongs to.

+   SC.View ownerForm

Discussion

May be null if the view does not belong to a form. This property is usually set automatically by an owner form view.

validateSubmit [R/W]

Invoked by the owner form just before submission.

+   Boolean validateSubmit

Discussion

Override with your own method to commit any final changes after you perform validation.

The default implementation simply calls performValidateSubmit() and returns that value.

validator [R/W]

The validator for this field.

+   SC.Validator validator

Discussion

Set to a validator class or instance. If this points to a class, it will be instantiated when the validator is first used.

validatorObject [R/W]

Returns the validator object, if one has been created.

+   SC.Validator validatorObject

performValidate()

Attempts to validate the receiver.

+   (String) performValidate (partialChange)

Parameters

(Boolean)partialChange YES if this is a partial edit.

Returns

(String) SC.VALIDATE_OK, error, or SC.VALIDATE_NO_CHANGE

Discussion

Runs the validator and returns SC.VALIDATE_OK, SC.VALIDATE_NO_CHANGE, or an error object. If no validator is installed, this method will always return SC.VALIDATE_OK.
