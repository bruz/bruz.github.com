 11 /**
 12   CoreQuery is a simplified DOM manipulation library used internally by 
 13   SproutCore to find and edit DOM elements.  Outside of SproutCore, you 
 14   should generally use a more full-featured DOM library such as Prototype
 15   or jQuery.
 16   
 17   CoreQuery itself is a subset of jQuery with some additional plugins.  If
 18   you have jQuery already loaded when SproutCore loads, in fact, it will 
 19   replace CoreQuery with the full jQuery library and install any CoreQuery
 20   plugins, including support for the SC.Enumerable mixin.
 21   
 22   Much of this code is adapted from jQuery 1.2.6, which is available under an
 23   MIT license just like SproutCore.
 24   
 25   h1. Using CoreQuery
 26   
 27   You can work with CoreQuery much like you would work with jQuery.  The core
 28   manipulation object is exposed as SC.$.  To find some elements on the page
 29   you just pass in a selector like this:
 30   
 31   {{{
 32     var cq = SC.$('p');
 33   }}}
 34   
 35   The object returned from this call is a CoreQuery object that implements 
 36   SC.Enumerable as well as a number of other useful manipulation methods.  
 37   Often times we call this object the "matched set", because it usually an
 38   array of elements matching the selector key you passed.
 39   
 40   To work with the matched set, just call the various helper methods on it.
 41   Here are some of the more useful ones:
 42   
 43   {{{
 44     // change all of the text red
 45     cq.css('color','red');
 46     
 47     // hide/show the set
 48     cq.hide();  cq.show();
 49     
 50     // set the text content of the set
 51     cq.text("Hello World!");
 52     
 53   }}}
 54   
 55   Of course, you can also chain these methods, just like jQuery.  Here is 
 56   how you might find all of the headings in your page, change their text and
 57   color:
 58   
 59   {{{
 60     SC.$('h1').text('Hello World!').css('color','red');
 61   }}}
 62   
 63   h1. Using CoreQuery with Views
 64   
 65   Usually you will not want to just blindly edit the HTML content in your
 66   application.  Instead, you will use CoreQuery to update the portion of the
 67   page managed by your SC.View instances.  Every SC.View instance has a $()
 68   property just like SC.$().  The difference is that this function will start
 69   searching from the root of the view.  For example, you could use the 
 70   following code in your updateDisplay method to set your content and color:
 71   
 72   {{{
 73     updateDisplay: function() {
 74       this.$().text(this.get('value')).css('color','red');
 75     }
 76   }}}
 77   
 78   You could also work on content within your view, for example this will 
 79   change the title on your view held in the span.title element:
 80   
 81   {{{
 82     updateDisplay: function() {
 83       this.$('span.title').text('Hello World');
 84       this.$().setClassName('sc-enabled', YES) ;
 85     }
 86   }}}
