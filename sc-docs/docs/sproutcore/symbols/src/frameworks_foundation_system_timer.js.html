  7 /**
  8   @class
  9 
 10   A Timer executes a method after a defined period of time.  Timers are 
 11   significantly more efficient than using setTimeout() or setInterval() 
 12   because they are cooperatively scheduled using the run loop.  Timers are
 13   also gauranteed to fire at the same time, making it far easier to keep 
 14   multiple timers in sync.
 15   
 16   h2. Overview
 17   
 18   Timers were created for SproutCore as a way to efficiently defer execution
 19   of code fragments for use in Animations, event handling, and other tasks.
 20   
 21   Browsers are typically fairly inconsistant about when they will fire a 
 22   timeout or interval based on what the browser is currently doing.  Timeouts 
 23   and intervals are also fairly expensive for a browser to execute, which 
 24   means if you schedule a large number of them it can quickly slow down the 
 25   browser considerably.
 26   
 27   Timers, on the other handle, are scheduled cooperatively using the 
 28   SC.RunLoop, which uses exactly one timeout to fire itself when needed and 
 29   then executes by timers that need to fire on its own.  This approach can
 30   be many timers faster than using timers and gaurantees that timers scheduled
 31   to execute at the same time generally will do so, keeping animations and
 32   other operations in sync.
 33   
 34   h2. Scheduling a Timer
 35 
 36   To schedule a basic timer, you can simply call SC.Timer.schedule() with 
 37   a target and action you wish to have invoked:
 38   
 39   {{{
 40     var timer = SC.Timer.schedule({ 
 41       target: myObject, action: 'timerFired', interval: 100 
 42     });
 43   }}}
 44 
 45   When this timer fires, it will call the timerFired() method on myObject.
 46   
 47   In addition to calling a method on a particular object, you can also use
 48   a timer to execute a variety of other types of code:
 49   
 50   - If you include an action name, but not a target object, then the action will be passed down the responder chain.
 51   - If you include a property path for the action property (e.g. 'MyApp.someController.someMethod'), then the method you name will be executed.
 52   - If you include a function in the action property, then the function will be executed.  If you also include a target object, the function will be called with this set to the target object.
 53 
 54   In general these properties are read-only.  Changing an interval, target,
 55   or action after creating a timer will have an unknown effect.
 56 
 57   h2. Scheduling Repeating Timers
 58   
 59   In addition to scheduling one time timers, you can also schedule timers to
 60   execute periodically until some termination date.  You make a timer
 61   repeating by adding the repeats: YES property:
 62   
 63   {{{
 64     var timer = SC.Timer.schedule({
 65       target: myObject, 
 66       action: 'updateAnimation', 
 67       interval: 100,
 68       repeats: YES, 
 69       until: Time.now() + 1000
 70     }) ;
 71   }}}
 72   
 73   The above example will execute the myObject.updateAnimation() every 100msec
 74   for 1 second from the current time.  
 75   
 76   If you want a timer to repeat without expiration, you can simply omit the
 77   until: property.  The timer will then repeat until you invalidate it.
 78   
 79   h2. Pausing and Invalidating Timers
 80   
 81   If you have created a timer but you no longer want it to execute, you can
 82   call the invalidate() method on it.  This will remove the timer from the 
 83   run loop and clear certain properties so that it will not run again.
 84   
 85   You can use the invalidate() method on both repeating and one-time timers.
 86   
 87   If you do not want to invalidate a timer completely but you just want to
 88   stop the timer from execution temporarily, you can alternatively set the
 89   isPaused property to YES:
 90   
 91   {{{
 92     timer.set('isPaused', YES) ;
 93     // Perform some critical function; timer will not execute
 94     timer.set('isPaused', NO) ;
 95   }}}
 96   
 97   When a timer is paused, it will be scheduled and will fire like normal, 
 98   but it will not actually execute the action method when it fires.  For a 
 99   one time timer, this means that if you have the timer paused when it fires,
100   it may never actually execute the action method.  For repeating timers, 
101   this means the timer will remain scheduled but simply will not execute its
102   action while the timer is paused.
103   
104   h2. Firing Timers
105   
106   If you need a timer to execute immediately, you can always call the fire()
107   method yourself.  This will execute the timer action, if the timer is not
108   paused.  For a one time timer, it will also invalidate the timer and remove
109   it from the run loop.  Repeating timers can be fired anytime and it will
110   not interrupt their regular scheduled times.
111 
112   
113   @extends SC.Object
114   @author Charles Jolley
115   @version 1.0
116   @since version 1.0
117 */
118 SC.Timer = SC.Object.extend(
119 /** @scope SC.Timer.prototype */ {
120 
121   /**
122     The target object whose method will be invoked when the time fires.
123     
124     You can set either a target/action property or you can pass a specific
125     method.
126     
127     @type {Object}
128     @field
129   */
130   target: null,
131   
132   /**
133     The action to execute.
134     
135     The action can be a method name, a property path, or a function.  If you
136     pass a method name, it will be invoked on the target object or it will 
137     be called up the responder chain if target is null.  If you pass a 
138     property path and it resolves to a function then the function will be 
139     called.  If you pass a function instead, then the function will be 
140     called in the context of the target object.
141     
142     @type {String, Function}
143   */
144   action: null,
145   
146   /**
147     Set if the timer should be created from a memory pool.  Normally you will
148     want to leave this set, but if you plan to use bindings or observers with
149     this timer, then you must set isPooled to NO to avoid reusing your timer.
150     
151     @property {Boolean}
152   */
153   isPooled: NO,
154   
155   /**
156     The time interval in milliseconds.
157     
158     You generally set this when you create the timer.  If you do not set it
159     then the timer will fire as soon as possible in the next run loop.
160     
161     @type {Number}
162   */
163   interval: 0,
164   
165   /**
166     Timer start date offset.
167     
168     The start date determines when the timer will be scheduled.  The first
169     time the timer fires will be interval milliseconds after the start 
170     date. 
171     
172     Generally you will not set this property yourself.  Instead it will be 
173     set automatically to the current run loop start date when you schedule 
174     the timer.  This ensures that all timers scheduled in the same run loop
175     cycle will execute in the sync with one another.
176     
177     The value of this property is an offset like what you get if you call
178     Date.now().
179     
180     @type {Number}
181   */
182   startTime: null,
183   
184   /**
185     YES if you want the timer to execute repeatedly.
186     
187     @type {Boolean}
188   */
189   repeats: NO,
190   
191   /**
192     Last date when the timer will execute.
193     
194     If you have set repeats to YES, then you can also set this property to
195     have the timer automatically stop executing past a certain date.
196     
197     This property should contain an offset value like startOffset.  However if
198     you set it to a Date object on create, it will be converted to an offset
199     for you.
200     
201     If this property is null, then the timer will continue to repeat until you
202     call invalidate().
203     
204     @type {Date, Number}
205   */
206   until: null,
207   
208   /**
209     Set to YES to pause the timer.
210     
211     Pausing a timer does not remove it from the run loop, but it will 
212     temporarily suspend it from firing.  You should use this property if
213     you will want the timer to fire again the future, but you want to prevent
214     it from firing temporarily.
215     
216     If you are done with a timer, you should call invalidate() instead of 
217     setting this property.
218     
219     @type {Boolean}
220   */
221   isPaused: NO,
222 
223   /**
224     YES onces the timer has been scheduled for the first time.
225   */
226   isScheduled: NO,
227   
228   /**
229     YES if the timer can still execute.
230     
231     This read only property will return YES as long as the timer may possibly
232     fire again in the future.  Once a timer has become invalid, it cannot 
233     become valid again. 
234     
235     @field
236     @type {Boolean}
237   */
238   isValid: YES,
239   
240   /**
241     Set to the current time when the timer last fired.  Used to find the 
242     next 'frame' to execute.
243   */
244   lastFireTime: 0,
245   
246   /**
247     Computed property returns the next time the timer should fire.  This 
248     property resets each time the timer fires.  Returns -1 if the timer 
249     cannot fire again.
250     
251     @property {Time}
252   */
253   fireTime: function() {
254     if (!this.get('isValid')) return -1 ;  // not valid - can't fire
255     
256     // can't fire w/o startTime (set when schedule() is called).
257     var start = this.get('startTime');
258     if (!start || start === 0) return -1; 
259 
260     // fire interval after start.
261     var interval = this.get('interval'), last = this.get('lastFireTime');
262     if (last < start) last = start; // first time to fire
263     
264     // find the next time to fire
265     var next ;
266     if (this.get('repeats')) {
267       if (interval === 0) { // 0 means fire as fast as possible.
268         next = last ; // time to fire immediately!
269         
270       // find the next full interval after start from last fire time.
271       } else {
272         next = start + (Math.floor((last - start) / interval)+1)*interval;
273       }
274       
275     // otherwise, fire only once interval after start
276     } else next = start + interval ;
277     
278     // can never have a fireTime after until
279     var until = this.get('until');
280     if (until && until>0 && next>until) next = until;
281     
282     return next ;
283   }.property('interval', 'startTime', 'repeats', 'until', 'isValid', 'lastFireTime').cacheable(),
284   
285   /**
286     Schedules the timer to execute in the runloop. 
287     
288     This method is called automatically if you create the timer using the
289     schedule() class method.  If you create the timer manually, you will
290     need to call this method yourself for the timer to execute.
291     
292     @returns {SC.Timer} The receiver
293   */
294   schedule: function() {
295     if (!this.get('isValid')) return this; // nothing to do
296     
297     this.beginPropertyChanges();
298     
299     // if start time was not set explicitly when the timer was created, 
300     // get it from the run loop.  This way timer scheduling will always
301     // occur in sync.
302     if (!this.startTime) this.set('startTime', SC.RunLoop.currentRunLoop.get('startTime')) ;
303 
304     // now schedule the timer if the last fire time was < the next valid 
305     // fire time.  The first time lastFireTime is 0, so this will always go.
306     var next = this.get('fireTime'), last = this.get('lastFireTime');
307     if (next >= last) {
308       this.set('isScheduled', YES);
309       SC.RunLoop.currentRunLoop.scheduleTimer(this, next);
310     }
311     
312     this.endPropertyChanges() ;
313     
314     return this ;
315   },
316   /**
317     Invalidates the timer so that it will not execute again.  If a timer has
318     been scheduled, it will be removed from the run loop immediately.
319     
320     @returns {SC.Timer} The receiver
321   */
322   invalidate: function() {
323     this.beginPropertyChanges();
324     this.set('isValid', NO);
325     SC.RunLoop.currentRunLoop.cancelTimer(this);
326     this.action = this.target = null ; // avoid memory leaks
327     this.endPropertyChanges();
328     
329     // return to pool...
330     if (this.get('isPooled')) SC.Timer.returnTimerToPool(this);
331     return this ;
332   },
333   
334   /**
335     Immediately fires the timer.
336     
337     If the timer is not-repeating, it will be invalidated.  If it is repeating
338     you can call this method without interrupting its normal schedule.
339     
340     @returns {void}
341   */
342   fire: function() {
343 
344     // this will cause the fireTime to recompute
345     var last = Date.now();
346     this.set('lastFireTime', last);
347 
348     var next = this.get('fireTime');
349 
350     // now perform the fire action unless paused.
351     if (!this.get('isPaused')) this.performAction() ;
352     
353      // reschedule the timer if needed...
354      if (next > last) {
355        this.schedule();
356      } else {
357        this.invalidate();
358      }
359   },
360 
361   /**
362     Actually fires the action. You can override this method if you need
363     to change how the timer fires its action.
364   */
365   performAction: function() {
366     var typeOfAction = SC.typeOf(this.action);
367     
368     // if the action is a function, just try to call it.
369     if (typeOfAction == SC.T_FUNCTION) {
370       this.action.call((this.target || this), this) ;
371 
372     // otherwise, action should be a string.  If it has a period, treat it
373     // like a property path.
374     } else if (typeOfAction === SC.T_STRING) {
375       if (this.action.indexOf('.') >= 0) {
376         var path = this.action.split('.') ;
377         var property = path.pop() ;
378 
379         var target = SC.objectForPropertyPath(path, window) ;
380         var action = target.get ? target.get(property) : target[property];
381         if (action && SC.typeOf(action) == SC.T_FUNCTION) {
382           action.call(target, this) ;
383         } else {
384           throw '%@: Timer could not find a function at %@'.fmt(this, this.action) ;
385         }
386 
387       // otherwise, try to execute action direction on target or send down
388       // responder chain.
389       } else {
390         SC.RootResponder.responder.sendAction(this.action, this.target, this);
391       }
392     }
393   },
394   
395   init: function() {
396     sc_super();
397     
398     // convert startTime and until to times if they are dates.
399     if (this.startTime instanceof Date) {
400       this.startTime = this.startTime.getTime() ;
401     }
402     
403     if (this.until instanceof Date) {
404       this.until = this.until.getTime() ;
405     }
406   },
407   
408   /** @private - Default values to reset reused timers to. */
409   RESET_DEFAULTS: {
410     target: null, action: null, 
411     isPooled: NO, isPaused: NO, isScheduled: NO, isValid: YES,
412     interval: 0, repeats: NO, until: null,
413     startTime: null, lastFireTime: 0
414   },
415   
416   /** 
417     Resets the timer settings with the new settings.  This is the method 
418     called by the Timer pool when a timer is reused.  You will not normally
419     call this method yourself, though you could override it if you need to 
420     reset additonal properties when a timer is reused.
421     
422     @params {Hash} props properties to copy over
423     @returns {SC.Timer} receiver
424   */
425   reset: function(props) {
426     if (!props) props = SC.EMPTY_HASH;
427     
428     // note: we copy these properties manually just to make them fast.  we 
429     // don't expect you to use observers on a timer object if you are using 
430     // pooling anyway so this won't matter.  Still notify of property change
431     // on fireTime to clear its cache.
432     this.propertyWillChange('fireTime');
433     var defaults = this.RESET_DEFAULTS ;
434     for(var key in defaults) {
435       if (!defaults.hasOwnProperty(key)) continue ; 
436       this[key] = SC.none(props[key]) ? defaults[key] : props[key];
437     }
438     this.propertyDidChange('fireTime');
439     return this ;
440   },

446   /** @private - removes the timer from its current timerQueue if needed. 
447     return value is the new "root" timer.
448   */
449   removeFromTimerQueue: function(timerQueueRoot) {
450     var prev = this._timerQueuePrevious, next = this._timerQueueNext ;
451 
452     if (!prev && !next && timerQueueRoot !== this) return timerQueueRoot ; // not in a queue...
453 
454     // else, patch up to remove...
455     if (prev) prev._timerQueueNext = next ;
456     if (next) next._timerQueuePrevious = prev ;
457     this._timerQueuePrevious = this._timerQueueNext = null ;
458     return (timerQueueRoot === this) ? next : timerQueueRoot ;
459   },
460   
461   /** @private - schedules the timer in the queue based on the runtime. */
462   scheduleInTimerQueue: function(timerQueueRoot, runTime) {
463     this._timerQueueRunTime = runTime ;
464     
465     // find the place to begin
466     var beforeNode = timerQueueRoot;
467     var afterNode = null ;
468     while(beforeNode && beforeNode._timerQueueRunTime < runTime) {
469       afterNode = beforeNode ;
470       beforeNode = beforeNode._timerQueueNext;
471     }
472 
473     if (afterNode) {
474       afterNode._timerQueueNext = this ;
475       this._timerQueuePrevious = afterNode ;
476     }
477     
478     if (beforeNode) {
479       beforeNode._timerQueuePrevious = this ;
480       this._timerQueueNext = beforeNode ;
481     }
482     
483     // I am the new root if beforeNode === root
484     return (beforeNode === timerQueueRoot) ? this : timerQueueRoot ;
485   },
486   
487   /** @private 
488     adds the receiver to the passed array of expired timers based on the 
489     current time and then recursively calls the next timer.  Returns the
490     first timer that is not expired.  This is faster than iterating through
491     the timers because it does some faster cleanup of the nodes.
492   */
493   collectExpiredTimers: function(timers, now) {
494     if (this._timerQueueRunTime > now) return this ; // not expired!
495     timers.push(this);  // add to queue.. fixup next. assume we are root.
496     var next = this._timerQueueNext ;
497     this._timerQueueNext = null;
498     if (next) next._timerQueuePrevious = null;
499     return next ? next.collectExpiredTimers(timers, now) : null; 
500   }

504 /** @scope SC.Timer */

506 /*  
507   Created a new timer with the passed properties and schedules it to 
508   execute.  This is the same as calling SC.Time.create({ props }).schedule().
509   
510   Note that unless you explicitly set isPooled to NO, this timer will be 
511   pulled from a shared memory pool of timers.  You cannot using bindings or
512   observers on these timers as they may be reused for future timers at any
513   time.
514   
515   @params {Hash} props Any properties you want to set on the timer.
516   @returns {SC.Timer} new timer instance.
517 */
518 SC.Timer.schedule = function(props) {
519   // get the timer.
520   var timer ;
521   if (!props || SC.none(props.isPooled) || props.isPooled) {
522     timer = this.timerFromPool(props);
523   } else timer = this.create(props);
524   return timer.schedule();
525 } ;
526 
527 /**
528   Returns a new timer from the timer pool, copying the passed properties onto
529   the timer instance.  If the timer pool is currently empty, this will return
530   a new instance.
531 */
532 SC.Timer.timerFromPool = function(props) {
533   var timers = this._timerPool;
534   if (!timers) timers = this._timerPool = [] ;
535   var timer = timers.pop();
536   if (!timer) timer = this.create();
537   return timer.reset(props) ;
538 };
539 
540 /** 
541   Returns a timer instance to the timer pool for later use.  This is done
542   automatically when a timer is invalidated if isPooled is YES.
543 */
544 SC.Timer.returnTimerToPool = function(timer) {
545   if (!this._timerPool) this._timerPool = [];
546 
547   this._timerPool.push(timer);
548   return this ;
549 };
