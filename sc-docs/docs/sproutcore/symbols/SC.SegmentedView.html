Class SC.SegmentedView

Extends: SC.View
Defined in: frameworks/sproutcore/frameworks/desktop/views/segmented.js
Availability: since SproutCore 1.0

Overview

SegmentedView is a special type of button that can display multiple segments. Each segment has a value assigned to it. When the user clicks on the segment, the value of that segment will become the new value of the control.

You can also optionally configure a target/action that will fire whenever the user clicks on an item. This will give your code an opportunity to take some action depending on the new value. (of course, you can always bind to the value as well, which is generally the preferred approach.)

Defining Your Segments

You define your segments by providing a items array, much like you provide to a RadioView. Your items array can be as simple as an array of strings or as complex as full model objects. Based on how you configure your itemKey properties, the segmented view will read the properties it needs from the array and construct the button.

You can define the following properties on objects you pass in:

| itemTitleKey | the title of the button | | itemValueKey | the value of the button | | itemWidthKey | the preferred width. if omitted, it autodetects | | itemIconKey | an icon | | itemActionKey | an optional action to fire when pressed | | itemTargetKey | an optional target for the action |

Property Details

allowsEmptySelection [R/W]

If YES, clicking a selected button again will deselect it, setting the segmented views value to null.

-   Object allowsEmptySelection

Discussion

Defaults to NO.

allowsMultipleSelection [R/W]

If YES, then clicking on a tab will not deselect the other segments, it will simply add or remove it from the selection.

-   Object allowsMultipleSelection

displayItems [R/W]

This computed property is generated from the items array based on the itemKey properties that you set.

-   Array displayItems

Discussion

The return value is an array of arrays that contain private information used by the SegmentedView to render.

You will not generally need to access or edit this property.

isEnabled [R/W]

Set to YES to enabled the segmented view, NO to disabled it.

-   Object isEnabled

itemActionKey [R/W]

The key that contains the action for this item.

-   String itemActionKey

Discussion

If defined, then selecting this item will fire the action in addition to changing the value. See also itemTargetKey.

itemIconKey [R/W]

The key that contains the icon for each item.

-   String itemIconKey

Discussion

If omitted, no icons will be displayed.

itemIsEnabledKey [R/W]

A key that determines if this item in particular is enabled.

-   String itemIsEnabledKey

Discussion

Note if the control in general is not enabled, no items will be enabled, even if the item's enabled property returns YES.

itemKeyEquivalentKey [R/W]

The key that contains the key equivalent for each item.

-   Object itemKeyEquivalentKey

Discussion

If defined then pressing that key equivalent will be like selecting the tab. Also, pressing the Alt or Option key for 3 seconds will display the key equivalent in the tab.

itemKeys [R/W]

The array of itemKeys that will be searched to build the displayItems array.

-   Array itemKeys

Discussion

This is used internally by the class. You will not generally need to access or edit this array.

items [R/W]

The array of items to display.

-   Array items

Discussion

This can be a simple array of strings, objects or hashes. If you pass objects or hashes, you must also set the various itemKey properties to tell the SegmentedView how to extract the information it needs.

itemTargetKey [R/W]

The key that contains the target for this item.

-   String itemTargetKey

Discussion

If this and itemActionKey are defined, then this will be the target of the action fired.

itemTitleKey [R/W]

The key that contains the title for each item.

-   String itemTitleKey

itemValueKey [R/W]

The key that contains the value for each item.

-   String itemValueKey

itemWidthKey [R/W]

The key that contains the desired width for each item.

-   String itemWidthKey

Discussion

If omitted, the width will autosize.

layoutDirection [R/W]

Change the layout direction to make this a vertical set of tabs instead of horizontal ones.

-   Object layoutDirection

localize [R/W]

If YES, titles will be localized before display.

-   Object localize

value [R/W]

The value of the segmented view.

-   Object value

Discussion

The SegmentedView's value will always be the value of the currently selected button. Setting this value will change the selected button. If you set this value to something that has no matching button, then no buttons will be selected.
