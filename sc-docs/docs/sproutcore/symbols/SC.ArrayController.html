Class SC.ArrayController

Extends: SC.Array, SC.Controller, SC.SelectionSupport
Defined in: frameworks/sproutcore/frameworks/foundation/controllers/array.js
Availability: since SproutCore 1.0
Author: Charles Jolley

Overview

An ArrayController provides a way for you to publish an array of objects for CollectionView or other controllers to work with. To work with an ArrayController, set the content property to the array you want the controller to manage. Then work directly with the controller object as if it were the array itself.

When you want to display an array of objects in a CollectionView, bind the "arrangedObjects" of the array controller to the CollectionView's "content" property. This will automatically display the array in the collection view.

Property Details

allowsSingleContent [R/W]

Set to YES if you want the controller to wrap non-enumerable content in an array and publish it.

-   Boolean allowsSingleContent

Discussion

Otherwise, it will treat single content like null content.

arrangedObjects [R/W]

Returns an SC.Array object suitable for use in a CollectionView.

-   SC.Array arrangedObjects

Discussion

Depending on how you have your ArrayController configured, this property may be one of several different values.

canAddContent [R/W]

Computed property insides whether you can add content.

-   Boolean canAddContent

Discussion

You can add content as long as the controller isEditable and the content is not a single object.

Note that the only way to simply add object to an ArrayController is to use the addObject() or pushObject() methods. All other methods imply reordering and will fail.

canRemoveContent [R/W]

Computed property indicates whether or not the array controller can remove content.

-   Boolean canRemoveContent

Discussion

You can delete content only if the content is not single content and isEditable is YES.

canReorderContent [R/W]

Computed property indicates whether you can reorder content.

-   Boolean canReorderContent

Discussion

You can reorder content as long a the controller isEditable and the content is a real SC.Array-like object. You cannot reorder content when orderBy is non-null.

content [R/W]

The content array managed by this controller.

-   SC.Array content

Discussion

You can set the content of the ArrayController to any object that implements SC.Array or SC.Enumerable. If you set the content to an object that implements SC.Enumerable only, you must also set the orderBy property so that the ArrayController can order the enumerable for you.

If you set the content to a non-enumerable and non-array object, then the ArrayController will wrap the item in an array in an attempt to normalize the result.

destroyOnRemoval [R/W]

Set to YES if you want objects removed from the array to also be deleted.

-   Boolean destroyOnRemoval

Discussion

This is a convenient way to manage lists of items owned by a parent record object.

Note that even if this is set to NO, calling destroyObject() instead of removeObject() will still destroy the object in question as well as removing it from the parent array.

hasContent [R/W]

Set to YES if the controller has valid content that can be displayed, even an empty array.

-   Boolean hasContent

Discussion

Returns NO if the content is null or not enumerable and allowsSingleContent is NO.

isEditable [R/W]

Makes the array editable or not.

-   Boolean isEditable

Discussion

If this is set to NO, then any attempts at changing the array content itself will throw an exception.

length [R/W]

Compute the length of the array based on the observable content

-   Number length

orderBy [R/W]

Used to sort the array.

-   String|Array|Function orderBy

Discussion

If you set this property to a key name, array of key names, or a function, then then ArrayController will automatically reorder your content array to match the sort order. (If you set a function, the function will be used to sort).

Normally, you should only use this property if you set the content of the controller to an unordered enumerable such as SC.Set or SC.SelectionSet. In this case the orderBy property is required in order for the controller to property order the content for display.

If you set the content to an array, it is usually best to maintain the array in the proper order that you want to display things rather than using this method to order the array since it requires an extra processing step. You can use this orderBy property, however, for displaying smaller arrays of content.

Note that you can only to use addObject() to insert new objects into an array that is ordered. You cannot manually reorder or insert new objects into specific locations because the order is managed by this property instead.

If you pass a function, it should be suitable for use in compare().

status [R/W]

Returns the current status property for the content.

-   Number status

Discussion

If the content does not have a status property, returns SC.Record.READY.

Method Detail

addObject()

Adds an object to the array.

-   (SC.ArrayController) addObject (object)

Parameters

(Object)object the object to add

Returns

(SC.ArrayController) receiver

Discussion

If the content is ordered, this will add the object to the end of the content array. The content is not ordered, the location depends on the implementation of the content.

If the source content does not support adding an object, then this method will throw an exception.

removeObject()

Removes the passed object from the array.

-   (SC.ArrayController) removeObject (object)

Parameters

(Object)object the object to remove

Returns

(SC.ArrayController) receiver

Discussion

If the underyling content is a single object, then this simply sets the content to null. Otherwise it will call removeObject() on the content.

Also, if destroyOnRemoval is YES, this will actually destroy the object.
